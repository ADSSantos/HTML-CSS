NOTA: GET: O método GET envia os dados do formulário como parte da URL. Os dados são anexados à URL na forma de parâmetros de consulta (query parameters). Limitação de tamanho: a quantidade de dados que podem ser enviados é limitada, geralmente limitada a alguns kilobytes. Os dados são visíveis na barra de endereços do navegador. Normalmente usado para enviar dados que não precisam ser seguros, como pesquisas ou solicitações de página. É mais eficiente para transferência de dados pequenos.
.

POST: O método POST envia os dados do formulário no corpo da requisição HTTP. Os dados não são visíveis na URL, tornando-os mais seguros para enviar informações sensíveis, como senhas. Não há limite específico de tamanho para os dados enviados pelo método POST. Os dados não são visíveis na barra de endereços do navegador. Normalmente usado para enviar dados sensíveis ou grandes, como formulários de login, upload de arquivos, etc. É menos eficiente que o método GET para transferência de dados pequenos, mas é mais seguro e mais apropriado para transferência de dados sensíveis.
